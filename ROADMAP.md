# Blazwind Roadmap (Q2 2026 Release)

This roadmap focuses on making Blazwind secure, intuitive, and fully documented by Q2 2026.

## Goals

- **Security:** Harden JS interop, dependency hygiene, and documentation on safe usage.
- **Clarity:** Improve architecture docs and contributor guidance.
- **Documentation:** Ensure each component and service has consistent docs and API references.

## Q4 2025 (Stabilization & Security)

- [ ] Complete security pass on JS interop (toast/dialog/workflow rendering).
- [ ] Add centralized input sanitization guidance and examples.
- [ ] Document CSP-friendly setup for the docs site.
- [ ] Review dependency vulnerability reports (monthly).

## Q1 2026 (Documentation & API Coverage)

- [ ] Ensure every `Services/*.cs` has matching `Services/Docs/*.md`.
- [ ] Add API reference for each major component group (Input, DataGrid, Map, Chart).
- [ ] Document build, release, and CI flows in a single "Maintainer Guide".

## Q2 2026 (Release Readiness)

- [ ] Expand unit tests for services and JS interop behaviors.
- [ ] Run a focused accessibility review for UI components.
- [ ] Publish migration guide from alpha to v1 release.
